### CameraMovementService 연산 작업의 필요성 분석

`moveCameraCinematic` 함수 내의 복잡한 연산은 **메타데이터(설정값)를 실제 3D 공간의 위치 정보로 변환**하기 위한 필수 과정입니다.

#### 1. 왜 메타데이터만으로 동작하지 않나요?
메타데이터는 **"상대적인 규칙"**을 정의하고, 코드는 이를 **"절대적인 좌표"**로 바꿉니다.

| 메타데이터 항목 | 코드에서의 연산 역할 |
| :--- | :--- |
| **Target Node** | 해당 노드가 3D 월드 상의 정확히 어느 좌표(`x, y, z`)에 있는지 실시간 계산 |
| **Distance (0.15)** | 타겟 중심점에서 방향 벡터만큼 곱해 실제 카메라가 서 있을 `Point` 계산 |
| **Direction (0, -1, 0)** | 카메라가 타겟을 바라보는 "각도"를 벡터 수학으로 구현 |

#### 2. 구체적인 연산 이유
- **Bounding Box 계산**: 노드의 크기와 모양이 제각각이므로, 카메라가 정확히 "물체의 정중앙"을 바라보게 하기 위해 매번 계산이 필요합니다.
- **시작 위치(Start Pos)**: 현재 카메라가 어디에 있든 상관없이 목적지까지 부드럽게 이동시키기 위해 현재 위치를 파악해야 합니다.
- **Bezier Curve (곡선 이동)**: 직선으로 딱딱하게 이동하는 대신, "시네마틱"한 부드러운 곡선 경로를 만들기 위해 제어점(Control Point)을 계산합니다.

#### 3. 요약
사용자님이 설정하신 `assembly-offsets.json`의 속성들은 이 **연산 작업의 "입력값"**으로 사용됩니다. 즉, 연산 로직이 있어야만 비로소 JSON에 적힌 `0.15`라는 숫자가 실제 3D 세상에서의 `15cm 거리`로 구현되는 것입니다. 동작 방식을 아주 정확하게 이해하고 계신 것이며, 그 구현을 위해 수반되는 필수적인 과정이라고 보시면 됩니다.
